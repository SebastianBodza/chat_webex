Each adapter handles webhook verification, message parsing, and API calls for a specific platform. Install only the adapters you need.

## Slack

```bash
pnpm add @chat-adapter/slack
```

### Single-workspace mode

For bots deployed to a single Slack workspace:

```typescript
import { createSlackAdapter } from "@chat-adapter/slack";

createSlackAdapter({
  botToken: process.env.SLACK_BOT_TOKEN!,
  signingSecret: process.env.SLACK_SIGNING_SECRET!,
});
```

### Multi-workspace mode

For distributed Slack apps installed across multiple workspaces via OAuth:

```typescript
createSlackAdapter({
  clientId: process.env.SLACK_CLIENT_ID!,
  clientSecret: process.env.SLACK_CLIENT_SECRET!,
  signingSecret: process.env.SLACK_SIGNING_SECRET!,
  encryptionKey: process.env.SLACK_ENCRYPTION_KEY,
});
```

Multi-workspace mode stores encrypted OAuth tokens in your state adapter (Redis). You need to set up OAuth install routes â€” see the example app for a full implementation.

### Setup

1. Create a Slack app at [api.slack.com/apps](https://api.slack.com/apps)
2. Enable **Event Subscriptions** with request URL: `https://your-domain.com/api/webhooks/slack`
3. Subscribe to bot events: `app_mention`, `message.channels`, `message.groups`, `message.im`, `message.mpim`, `reaction_added`, `reaction_removed`
4. Enable **Interactivity** with the same request URL (for buttons and modals)
5. Add OAuth scopes: `app_mentions:read`, `channels:history`, `chat:write`, `groups:history`, `im:history`, `mpim:history`, `reactions:read`, `reactions:write`, `users:read`
6. Install to your workspace

### Environment variables

| Variable | Description |
|----------|-------------|
| `SLACK_BOT_TOKEN` | Bot User OAuth Token (`xoxb-...`) |
| `SLACK_SIGNING_SECRET` | App signing secret for webhook verification |
| `SLACK_CLIENT_ID` | OAuth client ID (multi-workspace mode) |
| `SLACK_CLIENT_SECRET` | OAuth client secret (multi-workspace mode) |
| `SLACK_ENCRYPTION_KEY` | AES key for encrypting stored tokens (multi-workspace mode) |

### Features

| Feature | Supported |
|---------|-----------|
| Mentions | Yes |
| Reactions (add/remove) | Yes |
| Cards (Block Kit) | Yes |
| Modals | Yes |
| Streaming | Native API |
| DMs | Yes |
| Ephemeral messages | Yes (native) |
| File uploads | Yes |
| Typing indicator | Yes |
| Message history | Yes |

## Microsoft Teams

```bash
pnpm add @chat-adapter/teams
```

```typescript
import { createTeamsAdapter } from "@chat-adapter/teams";

createTeamsAdapter({
  appId: process.env.TEAMS_APP_ID!,
  appPassword: process.env.TEAMS_APP_PASSWORD!,
  appType: "SingleTenant",
  appTenantId: process.env.TEAMS_APP_TENANT_ID!,
});
```

### Setup

1. Create an Azure Bot resource in the [Azure Portal](https://portal.azure.com)
2. Configure the messaging endpoint: `https://your-domain.com/api/webhooks/teams`
3. Enable the Microsoft Teams channel
4. Create or note the App ID and password (client secret)
5. Create a Teams app manifest and install it in your organization

### Environment variables

| Variable | Description |
|----------|-------------|
| `TEAMS_APP_ID` | Azure Bot app ID |
| `TEAMS_APP_PASSWORD` | Azure Bot client secret |
| `TEAMS_APP_TENANT_ID` | Azure AD tenant ID |

### Features

| Feature | Supported |
|---------|-----------|
| Mentions | Yes |
| Reactions (add/remove) | Read-only |
| Cards (Adaptive Cards) | Yes |
| Modals | No |
| Streaming | Post+Edit fallback |
| DMs | Yes |
| Ephemeral messages | No (DM fallback) |
| File uploads | Yes |
| Typing indicator | No |
| Message history | No |

## Google Chat

```bash
pnpm add @chat-adapter/gchat
```

```typescript
import { createGoogleChatAdapter } from "@chat-adapter/gchat";

createGoogleChatAdapter({
  credentials: JSON.parse(process.env.GOOGLE_CHAT_CREDENTIALS!),
});
```

### Optional: Pub/Sub for all messages

By default, Google Chat only sends webhooks for @mentions. To receive all messages in a space, configure Pub/Sub:

```typescript
createGoogleChatAdapter({
  credentials: JSON.parse(process.env.GOOGLE_CHAT_CREDENTIALS!),
  pubsubTopic: process.env.GOOGLE_CHAT_PUBSUB_TOPIC,
  impersonateUser: process.env.GOOGLE_CHAT_IMPERSONATE_USER,
});
```

### Setup

1. Create a GCP project and enable the Google Chat API
2. Create a service account and download the JSON credentials
3. Configure the Chat app in the Google Cloud Console
4. Set the app URL to `https://your-domain.com/api/webhooks/gchat`
5. (Optional) Set up Pub/Sub topic and domain-wide delegation for full message access

### Environment variables

| Variable | Description |
|----------|-------------|
| `GOOGLE_CHAT_CREDENTIALS` | Service account JSON (as a string) |
| `GOOGLE_CHAT_PUBSUB_TOPIC` | Pub/Sub topic for receiving all messages (optional) |
| `GOOGLE_CHAT_IMPERSONATE_USER` | Admin email for domain-wide delegation (optional) |

### Features

| Feature | Supported |

| Streaming | Post+Edit fallback |
| DMs | Yes |
| Ephemeral messages | Yes (native) |
| File uploads | Yes |
| Typing indicator | No |

## Discord

```bash
pnpm add @chat-adapter/discord
```

```typescript
import { createDiscordAdapter } from "@chat-adapter/discord";

createDiscordAdapter({
  botToken: process.env.DISCORD_BOT_TOKEN!,
  publicKey: process.env.DISCORD_PUBLIC_KEY!,
  applicationId: process.env.DISCORD_APPLICATION_ID!,
});
```

### Gateway connection

Discord uses HTTP Interactions for button clicks but requires a Gateway WebSocket connection for receiving messages. In serverless environments, use a cron job to maintain the connection:

```typescript title="app/api/discord/gateway/route.ts"
export async function GET(request: Request) {
  // Verify cron secret
  const authHeader = request.headers.get("authorization");
  if (authHeader !== `Bearer ${process.env.CRON_SECRET}`) {
    return new Response("Unauthorized", { status: 401 });
  }

  // Connect gateway (idempotent)
  await bot.adapters.discord?.connectGateway();
  return new Response("OK");
}
```

### Setup

1. Create an application at [discord.com/developers](https://discord.com/developers/applications)
2. Create a bot user and copy the token
3. Set the Interactions Endpoint URL to `https://your-domain.com/api/webhooks/discord`
4. Invite the bot to your server with appropriate permissions
5. (Optional) Configure the Gateway cron for message events

### Environment variables

| Variable | Description |
|----------|-------------|
| `DISCORD_BOT_TOKEN` | Bot token |
| `DISCORD_PUBLIC_KEY` | Application public key (for webhook verification) |
| `DISCORD_APPLICATION_ID` | Application ID |
| `DISCORD_MENTION_ROLE_IDS` | Comma-separated role IDs that trigger mention handlers (optional) |
| `CRON_SECRET` | Secret for Gateway cron endpoint (optional) |

### Features

| Streaming | Post+Edit fallback |

## GitHub

```bash
pnpm add @chat-adapter/github
```

The GitHub adapter treats issue and pull request comments as messages, and issues/PRs as threads.

```typescript
import { createGitHubAdapter } from "@chat-adapter/github";

// Personal Access Token auth
createGitHubAdapter({
  token: process.env.GITHUB_TOKEN!,
  webhookSecret: process.env.GITHUB_WEBHOOK_SECRET!,
});

// GitHub App auth (supports multi-tenant)
createGitHubAdapter({
  appId: process.env.GITHUB_APP_ID!,
  privateKey: process.env.GITHUB_PRIVATE_KEY!,
  webhookSecret: process.env.GITHUB_WEBHOOK_SECRET!,
});
```

## Linear

```bash
pnpm add @chat-adapter/linear
```

The Linear adapter treats issue comments as messages and issues as threads.

```typescript
import { createLinearAdapter } from "@chat-adapter/linear";

// API key auth
createLinearAdapter({
  apiKey: process.env.LINEAR_API_KEY!,
  webhookSecret: process.env.LINEAR_WEBHOOK_SECRET!,
});

// OAuth app auth
createLinearAdapter({
  clientId: process.env.LINEAR_CLIENT_ID!,
  clientSecret: process.env.LINEAR_CLIENT_SECRET!,
  webhookSecret: process.env.LINEAR_WEBHOOK_SECRET!,
});
```
